# ドキュメント概要

- `specs/`: 仕様駆動開発で参照するコアドキュメント群。以下の構成で整理されている。
  - `design/`: 詳細設計。`00-index.md` を入口に各セクション（システム全体像〜DB設計）を参照する。
  - `requirements/`: 要件定義。`index.md` から機能要件を確認し、必要に応じてサブファイルを追加する。
  - `tasks/`: 進行中タスクリスト。`index.md` を更新し、完了項目を `[x]` に変更する。
- `guides/`: 開発者向けガイドライン。コーディング規約や JSDoc 規約などを配置する。

各ディレクトリ内に README や索引を適宜追加し、参照性を高めること。
