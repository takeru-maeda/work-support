# バージョニングガイド

アプリケーションおよび関連パッケージのリリースバージョンは **セマンティックバージョニング (SemVer)** に従います。形式は `MAJOR.MINOR.PATCH` です。

- **MAJOR**: 後方互換性を破る変更を含む場合にインクリメントします。例: API 契約の破壊的変更、既存データのマイグレーションが必須になる変更。
- **MINOR**: 後方互換性を保ったまま新機能を追加した場合にインクリメントします。既存の利用者に影響を与えない機能追加が対象です。
- **PATCH**: バグ修正やパフォーマンス改善など、挙動を変えない軽微な変更でインクリメントします。

### 運用ルール

1. バージョン更新はリリース単位で行い、Git のタグや CHANGELOG に記録します。
2. 破壊的変更かどうか判断が曖昧な場合はレビュー時にチームで確認し、保守負荷を避けるために保守的に MAJOR を選択してください。
3. ブランチや PR の説明には対象のバージョンを明記すると、デプロイ計画を立てやすくなります。

今後の変更時もこのガイドに従い、リリースの粒度と互換性ポリシーを揃えてください。
